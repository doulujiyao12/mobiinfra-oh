# MobiInfer LLM Chat (MNN 大模型端侧智能体应用)

本项目是一个在鸿蒙 (HarmonyOS NEXT) 设备上运行端侧大语言模型（基于 MNN 推理引擎），并结合 PC 辅助端实现“多模态智能体操作 (视觉 UI 自动化 Agent)”的完整解决方案。

---

## 🏗 架构简介

系统由两个主要部分组成：
1. **鸿蒙 App 端 (MobiInfer LLM Chat)**
   - 运行端侧模型（MobiInfer LLM），提供离线聊天和指令推理。
   - 负责任务的下发与用户授权交互。
2. **PC 服务器端 (Python)**
   - **模型静态服务**：提供大模型权重文件的局域网下载 (`serve_model.py`)。
   - **HDC 服务端**：提供局域网内的 HDC 代理和前置状态检测 (`hdc_server.py`)。
   - **Agent 引擎**：负责获取手机截图、运行大模型规划、通过 `hmdriver2` 模拟点击滑动等自动化操作 (`harmony_agent.py`)。

   > **注意**： 每次需要更新mobiinfra-oh/entry/src/main/cpp/include 下面两个头文件，以及mobiinfra-oh/entry/libs/arm64-v8a/libMNN.so 动态链接库

## 🛠 一、准备与安装

### 1. 环境依赖
- **PC 端**：
  - Windows / macOS / Linux。
  - Python 3.8+。
  - 配置好鸿蒙系统 `hdc` 环境变量（确保在命令行可以直接执行 `hdc list targets`）。
  - 安装 Python 依赖：
    ```bash
    pip install Pillow hmdriver2
    ```
- **手机端**：
  - 升级到支持的 HarmonyOS 操作系统。
  - 在开发者选项中开启并获取 **“无线调试”** 的 IP 与端口（例如：`192.168.1.100:35729`）。

### 2. PC 服务器端启动
你需要启动两个服务端脚本（推荐开启两个终端环境）：

1. **大模型下载服务**（如果没有模型文件需要先下载）：
   ```bash
   # 进入模型所在目录，启动 HTTP 文件服务（默认 9123 端口）
   python serve_model.py
   ```
2. **HDC 代理服与执行后端**：
   ```bash
   # 在 PC 端运行 HDC 服务（默认 9124 端口），它负责拉起并守护 harmony_agent.py 进程
   python entry/src/main/python/hdc_server.py
   ```

### 3. App 编译与安装
1. 使用 **DevEco Studio** 打开本项目 (MnnLlmChat)。
2. 配置好自动签名。
3. 点击 **Run** 或 **Debug** 编译打包，将 App 安装至手机。

## 📱 二、App 使用指南

App 采用沉浸式重设计的扁平化与玻璃态 UI 卡片，包含底部的两个主要标签页：**控制台** 和 **网络与设置**。

### 📌 第一次使用的配置流程（网络与设置页）
如果你是初次打开该 App，请切换到 **“网络与设置”** 页，完成以下步骤：
1. **填写 PC 的 IP 配置**：
   - **模型下载 IP**：PC 模型拉取服务器，如 `http://192.168.1.50:9123`
   - **HDC 远控 IP**：PC HDC Server。如 `http://192.168.1.50:9124`
   - **无线调试 IP**：手机在局域网暴露的无线调试端口，如 `192.168.1.101:5555`
2. **连接验证与远端 HDC 清理**：
   - 点击 **[测试远端 HDC 连接]**。这会让 PC 斩断其他冗余设备连接，主动绑定你这台设备的无线调试端口。
3. **下载模型库**：
   - 依次点击 **[检查连接]** > **[下载全部]**，APP会把 MNN 模型文件下载到沙箱。
   - 模型请使用 `mobiinfer llmexport` 命令导出。
   - 将 `./entry/src/main/python/serve_model.py` 放在导出后的模型文件夹下。
   - 先配置正确的 **模型下载 IP**，再执行模型下载。
4. **调整线程(可选)**：
   - 根据手机芯片算力，修改 **线程数** 然后点击 **[修改 Config]**，可加速推理。

### 🎙 日常对话与 Agent 控制（控制台页）
完成上述设置且模型下载完毕后，切回 **“控制台”** 页面：
1. 点击 **[加载大模型]**，输出框显示 "ok" 视为加载成功（背景变灰锁定）。
2. **对话模式**：在输入框填写聊天内容，点击 **[发送]**，纯文本本地交互。
3. **智能体 (Agent) 自动化测试**：
   - 在输入框撰写任务，如 `去淘宝搜索手机，并加入购物车`。
   - 点击下方 **[启动 PC 控制后端 (HDC)]** 按钮，手机开启监听端口（9126）
   - 点击橘色长按钮 **[开始智能体执行 (输入框为任务)]**，此时 PC 端的 `harmony_agent.py` 会获取任务，自动并截屏、推理并操作手机！

## 🖱 三、按钮功能详解

### 「控制台」页面
* **发送**：将文本框内容发给大模型并输出回复（附带防触控缩放和思维呼吸灯动效）。
* **加载大模型**：将下载到本地的模型文件预加载到手机内存/NPU中（耗时几秒）。
* **单轮推理**：测试模型基本联通性的简易按钮（后台只发一句固定的"你好"）。
* **重置会话**：清空当前大模型的记忆与多轮对话的历史记录。
* **开始智能体执行 (输入框为任务)**：设定一个操作任务给系统，让 PC 端后续可以提取并开始自动化规划。
* **启动 PC 控制后端 (HDC)**：打通手机与 PC 端 Python 脚本的 9126 端口通信桥梁，触发真正的自动化流程执行。
* **自动同意 (多选框)**：勾选后，模型每次计算出“我要点击/滑动”等意图时，无需确认弹窗直接执行。若不勾选，每次高风险动作皆会弹窗供你二次确认。

### 「网络与设置」页面
* **使用 modelv2 / modelv3 / modelv4 文件夹隔离模型**：三者互斥，或者全部不选回到默认 `model` 目录；下载、加载、删除都会作用到当前选中的模型目录，方便做多版本 A/B 测试对比。
* **测试远端 HDC 连接**：测试 PC 的 `hdc_server.py` 是否正常，并在 PC 上自动连接这台手机的无线程调试端口，并踢出其他占用设备。
* **修改 / 查看 Config**：读写沙箱中 `config.json` 文件里的线程数等属性。
* **检查连接**：测试下载服务器是否可以访问，并拉取服务器的文件目录。
* **下载全部**：执行所有缺失 MNN 模型切片文件的异步下载。
* **删除模型**：清空手机端的模型缓存，用于释放空间或重新下载更新。
* **列出沙箱图片 / 清理截图缓存**：为了排查错误或者节约空间，一键清理或打印 Agent 运行时产生在手机里的调试屏幕截图。

## ❓ 四、常见问题与排错 (FAQ)

**如果... 端口冲突导致 `TCP Port listen failed at 9126` 怎么办？**
答：最新的代码已经自带防呆机制，但如果不幸复现，可通过执行手机控制台命令修复：切断重练无线调试，APP 触发 **[启动 PC 控制后端]** 时内部的 `hdc fport rm tcp:9126 tcp:9126` 会自动把残留摘除清理。

**如果... [测试远端 HDC 连接] 一直无响应？**
答：检查你的 PC 防火墙是否放行了 `9123` 与 `9124` 端口，验证手机端是否和 PC 端处于同一公共局域网。

**如果... 模型输出是一堆乱码或崩溃退出？**
答：可能是线程数设置过大或者本地沙箱内的 `.weight` 权重文件在下载中断层或不全。请尝试：
1. 于“网络与设置”页将线程数（如 8）调小至 4。
2. 点击 **[删除模型]** 后重新下载。

**如果... 智能体开始执行了，但是 PC 端拉取截图失败报错？**
答：确保手机并未锁屏，息屏状态下无法通过 HDC 获取图层。若反复报 "Fail"，请插拔尝试 USB 连接模式重新赋权一次调试信任。
